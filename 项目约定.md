# 项目约定

## GitHub 上传代码流程

当用户说：

```text
帮我上传当前代码到 GitHub
```

Codex 必须按以下流程执行：

1. 先执行 `git status`，检查当前是否有未提交修改。
2. 如果没有新修改，回复用户：当前没有需要上传的修改。
3. 如果有新修改，先询问用户“本次修改说明写什么？”。
4. 如果用户已经在请求中给出说明，例如“说明：调整 bg 掉落间隔”，则直接使用该说明作为 commit message，不再重复询问。
5. 获得本次修改说明后，执行：

```bash
git add -A
git commit -m "<本次修改说明>"
git push
```

6. 上传完成后，向用户报告：
   - 提交说明
   - commit id
   - push 是否成功
   - 当前分支和远程仓库

## 注意事项

1. 不要在没有询问“本次修改说明”的情况下直接提交。
2. 不要把 GitHub token、密码或其他凭据写入聊天、文件或提交。
3. 如果 `git push` 遇到认证问题，让用户在本机终端输入 token，不要求用户把 token 发给 Codex。
4. 如果工作区存在明显无关修改，先说明修改范围，再继续提交。
