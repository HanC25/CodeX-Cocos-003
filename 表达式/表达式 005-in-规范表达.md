# 步骤五：游戏封面

## 目标

在步骤一到步骤四已经完成的游戏主流程之前，新增游戏封面界面。

游戏启动后，首先显示封面界面。

玩家点击封面中的 `fmlab01` 或 `fmlab02` 后，封面隐藏，进入步骤一到步骤四已经完成的游戏主流程。

## 基础约束

1. 不破坏步骤一、步骤二、步骤三、步骤四已经确认的游戏主流程。
2. 游戏仍然使用 Cocos Creator 3.8.8。
3. 画布仍然为横屏 1280x720。
4. 封面必须铺满 1280x720 游戏画布。
5. 封面出现时，主游戏对象 yu1-yu4 和 bg1-bg22 不应提前显示或运行。
6. 点击开始区域后，封面整体隐藏，主游戏流程从开场掉落阶段重新开始。
7. Cocos Creator 控制台不能出现 missing script 或脚本挂载错误。
8. TypeScript 编译和 `main.scene` JSON 校验需要通过。

## 封面对象定义

封面包含以下对象：

1. `fmbk`
   资源：`CC-003/assets/art/封面/bk01.png`
   项目实际资源：`CC-003/assets/art/封面/bk01.jpg`
   说明：原始表达写作 `bk01.png`，实现时以项目实际存在的 `bk01.jpg` 为准。

2. `fmlab01`
   资源：`CC-003/assets/art/封面/tw.png`

3. `fmlab02`
   资源：`CC-003/assets/art/封面/sz.png`

4. `fmstart`
   资源：`CC-003/assets/art/封面/start.png`

5. `fmclose`
   资源：`CC-003/assets/art/封面/close.png`

## 封面显示规则

1. 游戏进入后，首先显示封面界面。
2. `fmbk` 作为封面背景，尺寸为 1280x720，位置居中，铺满整个画布。
3. `fmlab01`、`fmlab02`、`fmstart`、`fmclose` 显示在 `fmbk` 之上。
4. 封面对象应作为同一个封面层统一管理，便于整体显示和隐藏。
5. 封面层显示时，主游戏流程不启动。

## 封面布局规则

1. 封面整体布局参考 `ck.png`。
2. `fmbk` 居中铺满画布。
3. `fmlab01` 放在画面右侧偏上区域，作为主要标题或主视觉标识。
4. `fmlab02` 放在画面左上角区域，作为辅助标识。
5. `fmstart` 放在画面右下偏中区域，作为开始按钮视觉元素。
6. `fmclose` 放在画面右上角区域。
7. 所有封面元素不能被主游戏对象遮挡。

## 点击开始规则

1. 玩家点击 `fmlab01` 时，开始游戏。
2. 玩家点击 `fmlab02` 时，开始游戏。
3. 本步骤原始表达没有要求点击 `fmstart` 开始游戏，因此实现时 `fmstart` 先只作为视觉元素。
4. `fmclose` 本步骤只显示，不实现关闭功能。
5. 点击开始后，封面层隐藏。
6. 点击开始后，主游戏流程从步骤一的开场掉落阶段开始执行。
7. 为避免误触，开始点击只响应一次。

## 主流程启动规则

1. 游戏脚本启动后，不立即执行 `playOpeningDrop`。
2. 游戏脚本先显示封面，并隐藏 yu1-yu4 与 bg1-bg22。
3. 玩家点击 `fmlab01` 或 `fmlab02` 后，调用主流程启动逻辑。
4. 主流程启动时，需要显示 yu1-yu4，并按步骤一到步骤四的规则执行开场掉落、bg 循环掉落和交互。
5. bg1-bg22 仍然由原有 bg 掉落阶段控制显示，不应在封面关闭瞬间全部显示。

## 层级规则

1. 封面层需要显示在主游戏对象之上。
2. 封面显示时，`fmbk` 在封面层最底部。
3. `fmlab01`、`fmlab02`、`fmstart`、`fmclose` 在 `fmbk` 之上。
4. 封面隐藏后，不再遮挡主游戏对象。

## 验收标准

1. 场景中存在 `fmbk`、`fmlab01`、`fmlab02`、`fmstart`、`fmclose` 五个封面对象。
2. `fmbk` 使用项目实际资源 `CC-003/assets/art/封面/bk01.jpg`。
3. `fmlab01` 使用 `CC-003/assets/art/封面/tw.png`。
4. `fmlab02` 使用 `CC-003/assets/art/封面/sz.png`。
5. `fmstart` 使用 `CC-003/assets/art/封面/start.png`。
6. `fmclose` 使用 `CC-003/assets/art/封面/close.png`。
7. 游戏启动后首先显示封面。
8. 封面显示时，主游戏对象不提前显示或运行。
9. 点击 `fmlab01` 后，封面隐藏并开始游戏主流程。
10. 点击 `fmlab02` 后，封面隐藏并开始游戏主流程。
11. `fmstart` 显示为视觉开始按钮，但本步骤不要求点击它开始游戏。
12. `fmclose` 显示在右上角，但本步骤不要求关闭功能。
13. 点击开始只响应一次，不重复启动主流程。
14. 开始后，yu1-yu4 按步骤一到步骤四既有规则运行。
15. Cocos Creator 控制台没有 missing script 或脚本挂载错误。
16. TypeScript 编译通过。
17. `main.scene` JSON 解析通过。
