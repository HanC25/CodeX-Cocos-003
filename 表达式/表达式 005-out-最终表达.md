# 步骤五：游戏封面最终表达

## 最终目标

在步骤一到步骤四已经完成并验证的游戏主流程之前，新增游戏封面界面。

游戏进入后，首先显示封面。

玩家点击封面中的 `fmlab01` 或 `fmlab02` 后，封面整体隐藏，主游戏流程从步骤一的 yu1-yu4 开场掉落阶段开始执行。

## 基础继承

1. 游戏使用 Cocos Creator 3.8.8。
2. 游戏画布为横屏 1280x720。
3. 步骤一到步骤四已经确认的 yu1-yu4、bg1-bg22、图形分类、bg 掉落和点击交互规则保持不变。
4. 封面只作为主游戏流程之前的入口界面。
5. 封面显示期间，主游戏对象不提前显示。
6. 封面显示期间，主游戏流程不提前启动。
7. Cocos Creator 控制台不能出现 missing script 或脚本挂载错误。
8. `main.scene` JSON、资源 UUID、脚本字段挂载关系需要通过校验。

## 封面对象

封面包含 5 个对象：

1. `fmbk`
   使用资源：`CC-003/assets/art/封面/bk01.jpg`
   说明：原始表达写作 `bk01.png`，项目实际存在资源为 `bk01.jpg`，最终实现以实际存在资源为准。

2. `fmlab01`
   使用资源：`CC-003/assets/art/封面/tw.png`

3. `fmlab02`
   使用资源：`CC-003/assets/art/封面/sz.png`

4. `fmstart`
   使用资源：`CC-003/assets/art/封面/start.png`

5. `fmclose`
   使用资源：`CC-003/assets/art/封面/close.png`

## 封面布局

1. `fmbk` 作为封面背景，位于 Canvas 中心。
2. `fmbk` 尺寸为 1280x720，铺满整个游戏画布。
3. `fmlab01` 位于画面右侧偏上区域，作为主标题视觉元素。
4. `fmlab02` 位于画面左上角区域，作为辅助标识视觉元素。
5. `fmstart` 位于画面右下偏中区域，作为开始按钮视觉元素。
6. `fmclose` 位于画面右上角区域，作为关闭按钮视觉元素。
7. 五个封面对象均作为 Canvas 子节点存在。
8. 五个封面对象位于 Canvas 子节点列表末尾，保证封面显示在主游戏对象之上。

## 点击开始规则

1. 玩家点击 `fmlab01` 时，开始游戏。
2. 玩家点击 `fmlab02` 时，开始游戏。
3. `fmstart` 在本步骤中只作为视觉开始按钮，不作为点击开始入口。
4. `fmclose` 在本步骤中只作为视觉关闭按钮，不实现关闭功能。
5. 点击开始只响应一次。
6. 第一次有效点击后，封面整体隐藏。
7. 封面隐藏后，显示 yu1-yu4。
8. 封面隐藏后，调用步骤一到步骤四已经确认的主游戏流程。

## 主流程启动规则

1. 游戏脚本启动时，不直接执行 `playOpeningDrop`。
2. 游戏脚本启动时先注册主游戏点击输入和封面点击输入。
3. 游戏脚本启动时调用封面显示逻辑。
4. 封面显示逻辑会显示全部封面对象。
5. 封面显示逻辑会隐藏 yu1-yu4。
6. 封面显示逻辑会隐藏 bg1-bg22。
7. 点击 `fmlab01` 或 `fmlab02` 后，封面隐藏。
8. 点击 `fmlab01` 或 `fmlab02` 后，yu1-yu4 显示。
9. 点击 `fmlab01` 或 `fmlab02` 后，执行 `playOpeningDrop`，主流程从开场掉落开始。
10. bg1-bg22 不在封面关闭瞬间全部显示，仍由步骤三的 bg 掉落阶段控制。

## 最终表达与规范表达的区别

1. `规范表达` 记录了编码前对封面界面、对象、层级和点击入口的执行规则。
2. `最终表达` 记录最终确认采用的资源、节点、脚本字段、关键方法和测试结果。
3. 本步骤原始表达中 `fmbk` 资源写作 `bk01.png`，最终实现确认项目实际资源为 `bk01.jpg`。
4. 本步骤最终确认 `fmstart` 是视觉开始按钮，不作为点击开始入口。
5. 本步骤最终确认 `fmclose` 是视觉关闭按钮，不实现关闭功能。
6. 本步骤最终确认封面节点需要位于 Canvas 子节点末尾，确保封面层级高于主游戏对象。
7. 本步骤最终确认主游戏流程必须被封面阻塞，不能在封面显示期间提前启动。
