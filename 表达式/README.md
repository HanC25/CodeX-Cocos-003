# 表达式目录说明

本目录用于记录每一个步骤的需求表达演进。

从现在开始，每一个步骤固定保留 3 个表述：

1. `表达式 00X-in-原始表达.md`
   记录用户最初的需求描述。

2. `表达式 00X-in-规范表达.md`
   记录 Codex 协助用户整理后的规范需求描述，用于正式编码前确认。

3. `表达式 00X-out-最终表达.md`
   记录修正完 bug、验证完效果后，重新提炼出的最终需求描述。

## 统一术语

`原始表达`：用户最初输入的自然语言需求。

`规范表达`：编码前整理出的、可执行、可验证、尽量规避风险的需求描述。

`最终表达`：实现并修正问题后沉淀的稳定表达，用于后续复用和对照。

## 命名规则

步骤编号使用 3 位数字：

```text
001
002
003
```

文件名格式固定为：

```text
表达式 00X-in-原始表达.md
表达式 00X-in-规范表达.md
表达式 00X-out-最终表达.md
```

如果某个步骤还没有完成最终验证，可以暂时缺少 `out-最终表达`，但需要在统计文件中标记状态。

## 转换规范迭代规则

`表达式转换规范 00X.md` 不是永久不变的总规范，而是基于 `表达式 00X` 系列沉淀出来的阶段性规范产品。

完成某个表达式系列后，需要基于以下内容生成新的转换规范：

1. 上一版表达式转换规范。
2. 当前表达式系列的原始表达、规范表达、最终表达。
3. 当前实现与修正过程中新增的经验、风险、问题和稳定表述。

例如：

```text
表达式 001 系列
        ↓
表达式转换规范 001.md

表达式 002 系列
        ↓
表达式转换规范 002.md
```

`表达式转换规范 00X.md` 代表截至表达式 `00X` 为止，关于“原始表达 -> 规范表达 -> 最终表达”的最新方法论。
