# 自动演出部分：原始表达模板

本模板用于书写自动演出类步骤的 `原始表达`。

适用范围包括：

1. 游戏开始演出。
2. 开场细节调整。
3. 循环掉落演出。
4. 阶段衔接演出。
5. 其他不依赖玩家主动操作、由系统自动驱动的演出或运动规则。

注意：这是 `原始表达` 模板，不是 `规范表达` 模板。

`原始表达` 的目标是：清楚、完整、有顺序地记录需求。

工程实现口径、风险补充、验收标准细化，由后续 `规范表达` 阶段完成。

## 推荐填写顺序

以后书写自动演出类 `原始表达` 时，建议按以下顺序组织：

1. 阶段目标
2. 开始条件
3. 参与对象
4. 对象分类
5. 初始状态
6. 运动规则
7. 轨道 / 位置规则
8. 随机规则
9. 多对象调度规则
10. 动画 / 状态规则
11. 层级规则
12. 阶段结束 / 下一阶段
13. 补充说明

这个顺序的目的：

1. 先定义谁参与。
2. 再定义什么时候开始。
3. 然后定义怎么运动。
4. 最后定义怎么结束和如何衔接下一阶段。
