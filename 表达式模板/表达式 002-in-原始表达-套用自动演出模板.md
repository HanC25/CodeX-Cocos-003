# 表达式 002-in-原始表达：套用自动演出模板

## 步骤 002：调整步骤 1 的一些细节

## 一、阶段目标

本步骤要实现的自动演出是：

调整步骤 1 中 yu1、yu2、yu3、yu4 的缩放、轨道、随机掉落顺序、随机掉落时间、左右摆动、停止动作和停止后上下浮动。

本步骤属于：

开场细节调整。

## 二、阶段开始条件

本步骤从什么时候开始：

1. 触发条件：

步骤 1 的游戏开始画面基础上进行调整。

2. 是否需要等待：

原始表达未说明。

## 三、参与对象

1. 对象数量：

4 个。

2. 对象名称：

yu1、yu2、yu3、yu4。

3. 对象类型：

Spine。

4. 对象资源：

沿用步骤 1：

yu1 -> `assets/art/common/spine/11`

yu2 -> `assets/art/common/spine/22`

yu3 -> `assets/art/common/spine/33`

yu4 -> `assets/art/common/spine/44`

## 四、对象分类

本步骤不涉及对象分类。

## 五、初始状态

1. 是否可见：

沿用步骤 1。

2. 初始位置：

沿用步骤 1。

3. 初始缩放：

yu1～yu4 缩放到现有大小的 80%。

4. 初始动作 / 状态：

掉落中播放 `1_zhangzui`。

## 六、运动规则

1. 运动方式：

掉落、左右摆动、停止后上下浮动。

2. 运动方向：

yu1～yu4 沿 z1～z4 掉落。

3. 运动速度 / 时间：

每一个对象掉落时间随机在 3～6 秒。

停止后上下浮动频次随机在 2 次 / 3 秒～3 次 / 3 秒。

4. 运动是否循环：

掉落不循环。

停止后的上下浮动循环。

## 七、轨道 / 位置规则

1. 是否使用轨道：

使用。

2. 轨道名称：

z1、z2、z3、z4。

3. 轨道定义：

画布水平 4 等分后，从左到右，4 个等分区域的中轴分别定义为 z1、z2、z3、z4。

4. 对象和轨道的对应关系：

yu1～yu4 沿着 z1～z4 掉落。

5. 停止位置 / 目标位置：

沿用步骤 1 的停止位置。

## 八、随机规则

1. 是否有随机：

有随机。

2. 随机内容：

随机掉落间隔、随机掉落顺序、随机掉落时间、随机摆动幅度、随机摆动频次、随机上下浮动幅度、随机上下浮动频次。

3. 随机范围：

随机项 -> 范围

掉落间隔 -> 0.3～0.6 秒

掉落时间 -> 3～6 秒

左右摆动幅度 -> 5～30 像素

左右摆动频次 -> 3～6 次

上下浮动幅度 -> 5～20 像素

上下浮动频次 -> 2 次 / 3 秒～3 次 / 3 秒

4. 随机频率：

掉落顺序为随机顺序。

其他随机频率原始表达未明确说明。

## 九、多对象调度规则

1. 是否同时开始：

不同时开始。

2. 对象之间的时间间隔：

3. 是否允许多个对象同时出现在画面中：

4. 是否有同轨道 / 同位置限制：

yu1～yu4 分别沿 z1～z4 掉落。
